# 风险预警功能需求文档

**版本**: v1.0
**日期**: 2026-02-05
**状态**: 已实现

---

## 一、功能概述

在医生端 H5 应用的患者管理模块中，新增风险预警功能，帮助医生快速识别和管理存在健康风险的患者。功能包含三个核心模块：风险统计仪表盘、风险详情二级页、患者列表增强。

---

## 二、功能模块

### 2.1 风险统计仪表盘 (Risk Dashboard)

#### 功能描述
在患者列表页顶部新增三个独立统计卡片，分别对应三类健康指标的预警统计。

#### 卡片类型

| 卡片名称 | 指标类型 | 标识符 |
|---------|---------|--------|
| 血糖 | Glucose | `glucose` |
| 血压 | Blood Pressure | `bp` |
| 心率 | Heart Rate | `hr` |

#### 数据维度

每个卡片展示两个核心数据：

1. **当日预警数**
   - 定义：今天（00:00 至当前时间）产生预警的患者总数
   - 统计方式：去重统计（同一患者当日多次预警只计 1 人）
   - 用途：反映当天的即时风险状况

2. **近 7 日预警数**
   - 定义：过去 7 天内产生预警的总人次
   - 统计方式：累计统计（同一患者多次预警计多次）
   - 用途：体现风险趋势，帮助医生判断整体健康管理效果

#### 视觉设计要求

| 要求项 | 规格说明 |
|-------|---------|
| 整体风格 | 科技感、现代简洁 |
| 背景效果 | 微透明渐变（半透明叠加） |
| 血糖卡片配色 | 琥珀色系 `bg-amber-50/60` |
| 血压卡片配色 | 红色系 `bg-red-50/60` |
| 心率卡片配色 | 紫色系 `bg-purple-50/60` |
| 卡片间距 | `gap-3`（12px） |
| 卡片圆角 | 14px |
| 边框 | 1px 微透明同色系边框 |

#### 交互行为

- **点击交互**: 点击任一卡片，跳转至对应类型的【风险详情二级页】
- **下拉刷新**: 支持下拉刷新更新统计数据
- **加载状态**: 数据加载时显示骨架屏或加载动画

### 2.2 风险详情二级页 (Risk Detail Sub-page)

#### 功能描述
根据点击的卡片类型（血糖/血压/心率），展示该类型下的风险患者清单。

#### 页面结构

```
┌─────────────────────────────────┐
│  ← 返回    [页面标题]           │  导航栏
├─────────────────────────────────┤
│  ┌─────────────────────────┐   │
│  │  [图标]  近7日预警人次   │   │  统计摘要卡片
│  │          [数值]         │   │
│  └─────────────────────────┘   │
├─────────────────────────────────┤
│  预警患者清单    共 X 人        │  列表标题
├─────────────────────────────────┤
│  ┌─────────────────────────┐   │
│  │ [危险条] 患者信息卡片    │   │  患者卡片列表
│  └─────────────────────────┘   │
│  ┌─────────────────────────┐   │
│  │ [危险条] 患者信息卡片    │   │
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

#### 患者信息展示

每个患者卡片需展示以下信息：

| 信息项 | 说明 | 示例 |
|-------|------|------|
| 基础信息 | 姓名、年龄、手机号 | 王小明 · 35岁 · 137****7001 |
| 预警统计 | 选定周期内的累计预警次数 | 3次/周 |
| 核心数值 | 最近一次超标的异常数值 | 158/98 mmHg |
| 预警时间 | 最近预警的相对时间 | 2小时前 |
| 预警描述 | 系统生成的预警说明（可选） | 血压偏高，建议复测 |

#### 视觉标识

- **危险等级条**: 卡片左侧 4px 宽度的色条
  - 危险（danger）: 红色渐变 `#EF4444 → #DC2626`
  - 警告（warning）: 橙色渐变 `#F59E0B → #D97706`

- **预警次数徽章**: 右上角显示周预警次数
  - 危险级: 红色背景 `#FEE2E2` + 红色文字 `#DC2626`
  - 警告级: 黄色背景 `#FEF3C7` + 橙色文字 `#D97706`

#### 交互行为

- **电话拨打**: 点击电话图标直接拨打患者电话
- **返回导航**: 点击返回按钮返回患者列表页

### 2.3 患者列表增强 (List Identification)

#### 功能描述
在现有患者列表基础上，为存在风险预警的患者增加醒目的视觉标识和关键信息外露。

#### 优先级排序规则

```
排序优先级（从高到低）：
1. 有风险预警的患者 → 按预警次数降序
2. 无风险预警的患者 → 按原有排序（注册时间倒序）
```

#### 视觉标识系统

| 标识元素 | 位置 | 说明 |
|---------|------|------|
| 风险状态条 | 卡片左侧边缘 | 4px 宽度色条，红色(danger)/橙色(warning) |
| 预警铃铛图标 | 姓名右侧 | 红色铃铛图标，带轻微摇晃动画 |
| 预警徽章 | 卡片右上区域 | 显示 "X次预警"，红色背景 |
| 特殊头像 | 头像区域 | 风险患者使用红色渐变背景 |

#### 异常信息外露

在患者卡片中直接展示：

1. **风险类型标签**
   - 格式: 胶囊形标签
   - 内容: `[高血糖]` `[高血压]` `[心率异常]`
   - 配色:
     - 血糖: 琥珀色 `#FEF3C7` + `#B45309`
     - 血压: 红色 `#FEE2E2` + `#DC2626`
     - 心率: 紫色 `#F3E8FF` + `#7C3AED`

2. **异常数值**
   - 位置: 风险标签同行右侧
   - 格式: 等宽字体，红色高亮
   - 示例: `158/98 mmHg`

#### 卡片结构对比

**普通患者卡片:**
```
┌──────────────────────────────────────┐
│ [头像]  姓名 · 年龄  [已绑定]        │
│         137****7001      3天前       │
│                              [电话]  │
└──────────────────────────────────────┘
```

**风险患者卡片:**
```
┌──────────────────────────────────────┐
│ [红色状态条]                         │
│ [红头像] 姓名 🔔 · 年龄  [4次预警]   │
│         [高血压] [高血糖] 158/98     │
│         137****7001      今天        │
│                              [电话]  │
└──────────────────────────────────────┘
```

## 三、数据模型

### 3.1 预警记录实体 (HealthAlert)

```typescript
interface HealthAlert {
  id: number;                           // 主键
  patientId: number;                    // 患者 ID
  alertType: 'glucose' | 'bp' | 'hr';   // 预警类型
  value: string;                        // 异常数值，如 "158/98"
  unit: string;                         // 单位，如 "mmHg"
  level: 'warning' | 'danger';          // 预警级别
  description: string;                  // 预警描述
  alertTime: Date;                      // 预警时间
}
```

### 3.2 预警类型定义

| 类型 | 标识符 | 单位 | 数值格式示例 |
|-----|--------|------|-------------|
| 血糖 | `glucose` | mmol/L | 8.2 |
| 血压 | `bp` | mmHg | 158/98 |
| 心率 | `hr` | bpm | 112 |

### 3.3 预警级别定义

| 级别 | 标识符 | 视觉表现 | 触发条件（示例） |
|-----|--------|---------|----------------|
| 警告 | `warning` | 橙色系 | 血压 140-159/90-99 |
| 危险 | `danger` | 红色系 | 血压 ≥160/100 |

## 四、API 接口

### 4.1 获取风险仪表盘数据

```
GET /api/doctor/risk-dashboard
Authorization: Bearer <token>
```

**响应示例:**
```json
{
  "glucose": { "todayCount": 1, "weekCount": 4 },
  "bp": { "todayCount": 1, "weekCount": 4 },
  "hr": { "todayCount": 1, "weekCount": 2 }
}
```

### 4.2 获取风险患者列表

```
GET /api/doctor/risk-patients?type=bp
Authorization: Bearer <token>
```

**参数:**
- `type`: 预警类型，可选值 `glucose` | `bp` | `hr`

**响应示例:**
```json
[
  {
    "id": 1,
    "name": "王小明",
    "age": 35,
    "phone": "13700137001",
    "gender": "male",
    "totalAlertCount": 3,
    "weekAlertCount": 3,
    "latestValue": "158/98",
    "latestUnit": "mmHg",
    "latestLevel": "danger",
    "latestAlertTime": "2026-02-05T06:08:33.388Z",
    "description": "血压偏高，建议复测并调整用药"
  }
]
```

### 4.3 获取患者列表（增强版）

```
GET /api/doctor/patients
Authorization: Bearer <token>
```

**响应示例:**
```json
[
  {
    "id": 1,
    "name": "王小明",
    "gender": "male",
    "age": 35,
    "phone": "13700137001",
    "createdAt": "2026-02-05T08:08:33.000Z",
    "hasAlert": true,
    "alertCount": 4,
    "alertTypes": ["bp", "glucose"],
    "latestAlert": {
      "alertType": "bp",
      "value": "158/98",
      "unit": "mmHg",
      "level": "danger",
      "alertTime": "2026-02-05T06:08:33.388Z"
    }
  }
]
```
